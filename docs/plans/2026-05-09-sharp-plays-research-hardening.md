# PP-MCP Sharp Plays Research Hardening Plan

> For Hermes: Use subagent-driven-development if implementing this later. Keep changes small, test-first, and verify against the live dashboard on port 3477.

Goal: Make the sharp plays dashboard research queue reliable enough for fast bet/pass triage without false confidence from broken web auth, stale times, noisy agent output, or weak evidence.

Architecture: Keep the current server-rendered Node dashboard. Harden the research runner, add a lightweight health/debug surface, tighten the research prompt/output contract, and add deterministic validation around target-book price, match status, and queue process state.

Tech stack: Node.js commonjs, node:test, Hermes CLI --oneshot, Tavily web backend, terminal fallback, local JSON queue state under ~/.hermes/propprofessor/sharp-play-research/.

---

## Current state

Already fixed:
- Research agents now load ~/.hermes/.env into child env.
- If TAVILY_API_KEY is present, research agents use an isolated Hermes home with web backend forced to Tavily.
- Isolated config strips mcp_servers to avoid unrelated OAuth popups.
- auth.json is symlinked/copied into the isolated home so model auth still works.
- Tests pass: node --check and node --test test/propprofessor-sharp-plays-dashboard.test.js.
- Live dashboard returns 200 on /api/research and main page.

Still worth improving:
- Research can still infer target-book availability indirectly from search results instead of a first-party/live payload.
- Queue visibility is useful but not yet diagnostic enough when agents hang or web fallback degrades.
- There is no one-click requeue/clear-failed workflow from the dashboard.
- The isolated Hermes config is generated by regex text surgery; it works, but needs a focused regression around model/auth preservation and mcp removal.
- New dashboard files are still untracked in git, so the feature is not safely checkpointed.

## Acceptance criteria

This work is good enough when:
- Research agents reliably use Tavily when Firecrawl gateway is unauthorized.
- Dashboard visibly shows research env health.
- No background research job can get stuck silently.
- Failed/done research can be cleared or requeued without editing JSON.
- Bet verdicts distinguish direct target-book verification from indirect web evidence.
- Tennis stale verdicts require external match status, not PP start time only.
- Tests pass and the feature is committed/checkpointed.
